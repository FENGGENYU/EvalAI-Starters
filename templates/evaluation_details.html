Submission Data Format

Participants are required to generate a JSON file for submission, which should contain the predicted semantic label for each convex shape.

The generated JSON file should adhere to the following dictionary format, where each key in the dictionary represents the ASIN of one product, and the value represents the predicted labels of convex shapes. 


{
  "B075X33T21": [0, 1, 0, 2, 3],
  "B084DCJKSL": [0, 0, 0, 3, 3],
}
Participants need to submit one JSON file for development dataset and one JSON file for evaluation dataset. The order of items in the submitted JSON file is not required, but the convex labels of each ASIN should follow the same order as the convex shape meshes in the 'convex' folder.

Evaluation Metrics

The evaluation metric used for this challenge is the prediction accuracy. For each convex shape, the prediction accuracy will be determined by comparing the ground truth label with the predicted label. The overall prediction accuracy will be computed as the average accuracy across all convex shapes.
